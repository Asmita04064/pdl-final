# ResQNet – Offline Emergency Communication System

##  Problem
During disasters, internet fails. People cannot send SOS messages.

##  Existing Solutions Fail
- Require internet
- No real-time offline communication

##  Our Solution
A hybrid system using:
- Bluetooth / local mesh communication
- Cloud sync when available
- Real-time alert broadcasting

##  Features
- SOS alert system
- Location tagging (Google Maps)
- Offline-first architecture
- Real-time updates

##  Tech Stack
- Frontend: React
- Backend: Node / Firebase
- Maps: Google Maps API

##  Testing
- Unit tests 
- API testing
- Edge cases handled


##  Deployment
Live: https://pdl-final.vercel.app/

##  Future Scope
- Radio-based communication
- AI-based disaster prediction
